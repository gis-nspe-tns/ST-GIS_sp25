Preamble

This week’s Class 8 assignment will feature raster geoprocessing within a larger suitability model. Here you want to find suitable areas within the Northeastern Indian state of Assam between 800 and 1000 acres that meet a series of geographic criteria that are defined on raster surfaces.

The assignment features several sections as follows:

  • Rasterization of input features that have been derived from OpenStreetMap.
  • Development of various raster surfaces needed for proximity criteria.
  • Analysis of distance threshold criteria upon raster surfaces.
  • Final suitability combination layer.
  • Subset of ‘Most Suitable’ locations.
  • Cartographic design for the preceding raster analysis suitability model.

Assignment Step 2 - Data preprocessing for the vector features

This Multi-Criteria Evaluation (MCE) suitability assignment will conducted primarily in raster overlay. So first item of data preparation is to convert vector > raster via Processing Toolbox:

In this step, the vector features will be rasterized in preparation for a Euclidean Distance output which is a Global processing function:

To Start, input the roads layer and populate the Rasterize tool with the following parameters:

  • 1 as the A fixed value to burn
  • Geoferenced units as the Output raster size units
  • 60 as both Width/Horizontal resolution and Height/Vertical resolution
  • Output extent = boundary (choose boundary at far right of dialog box)
  • nodata set to Not set
  • save to exports directory as roads_raster_60m

While this process can be repeated layer by layer, via the Rasterize GDAL tool, batch processing can speed up this process. To start, access the Batch Process option at the bottom of any toolset:

The tool is then populated with the remaining layers (but not roads as that is done). Make sure to populate with the same parameters as roads run. At the far right of the tool dialog, there will be a button for the output location. Select the top button and populate with the autofill mode set to fill with parameter values and input layer. This will result in a hash value unique to each input that now will be exported to the chosen export location. Keep in mind that once the tool finishes, the results will not be loaded into the layers panel; they will be placed in the export location (and empty exports folder already exists for this location in the assignment data folder:

Warning

the 60 meter spatial resolution is a bit coarse; the upside is that the processing speed and memory space will be preserved. If desired, the spatial resolution can be increased towards 15 meters for each vector input. This will produce a more precise result to the processing.

Assignment Step 3 - Merge Water Themes

Given that water as polyline vs polygons are not a meaningful distinction for this analysis, it makes sense to merge the water themes. Here Polygons and Polylines for water really represent the same theme - water assets. So those can be merged via Raster Calculator and set to output as raster_water_merged_60m:

"water_polylines_raster_60m@1" + "water_polygons_raster_60m@1"

Before proceeding, one issue needs to be resolved. Where the polylines and polygons intersect, a value of 2 is produced (white). This value should really be simply 1. To correct this condition, simply re-run Raster Calculator, selecting all values greater than 0 to produce the value of 1:

"raster_water_merged_60m@1">0

Save the output as raster_water_60m.tif:

Assignment Step 4 - Proximity Distance Surfaces

In review of the original criteria we see that their are two distance criteria:

  • Close to roads
  • Away from water bodies

Distance criteria is handled well by the Raster Distance GDAL tool. Here a pixel value is assigned to every cell location relative to the theme input feature - road and water bodies, respectively. These resulting rasters can be then used to determine suitable areas which are within certain distances from the inputs.

First, Locate the tool via Processing Toolbox:

To start, input the roads_raster_60m.tif raster and populate the tool with the following parameters:

  • Georeferenced coordinates as the Distance units
  • 5000 (5 kilometers) as the Maximum distance
  • Nodata value to use for the destination proximity raster value is Not set
  • Output = roads_proximity.tif

Repeat the proximity distance process with input of the raster_water_60m.tif, same parameters:

  • Georeferenced coordinates as the Distance units
  • 5000 (5 kilometers) as the Maximum distance
  • Nodata value to use for the destination proximity raster value is Not set
  • Output = water_proximity.tif

In review of the result, overlay the water features atop water_proximity.tif. Change the Max value to 5000 in symbology for water_proximity.tif so that cell values are scaled better. Here we can see how distance from water is indeed the result of the tool:

Assignment Step 5 - Raster Reclassification

With the distance analysis complete, scores are applied based on thresholds. The following score schema will be used for roads_proximity.tif. This schema gives more points to less distance (closer to roads) across 3 discrete classes:

0-1000m –> 100

1000-5000m –> 50

5000m –> 10

Reclassification can occur several ways. Here use the Raster Calculator to weight to the three classes noted above using the following equation.

The following formula will be utilized in Raster Calculator with roads_proximity.tif as input and an output named roads_reclass.tif:

100*("roads_proximity@1"<=1000) + 50*("roads_proximity@1">1000)*("roads_proximity@1"<=5000) + 10*("roads_proximity@1">5000)

Repeat the re-classification process for the water_proximity.tif. Here the scheme will be reversed, where pixels that are further away from water shall have higher score:

0-1000m –> 10

1000 -5000m —> 50

5000m –> 100

The following formula will be utilized in Raster Calculator with water_proximity.tif as input and an output named water_reclass.tif:

100*("water_proximity@1">5000) + 50*("water_proximity@1">1000)*("water_proximity@1"<=5000) + 10*("water_proximity@1"<1000)

Assignment Step 6 - Suitability Combination

The next and final analysis process will produce the final suitability result. In the following formula, the reclass rasters and the protected regions raster are the factors that are multiplied to produce suitability results within the state boundary. By multiplying the product at the end by raster_boundary@1, cells outside the state boundary are discarded.

Output name = overlay.tif

("roads_reclass@1" + "water_reclass@1")*("protected_regions_raster_60m@1"  !=  1 )*"boundary_raster_60m@1"

Assignment Step 7 - Subsetting a Final Most Suitable Class

With the overaly suitability complete, there still remains one last ordinal class - the most suitable based on threshold criteria of 800 - 1000 continuous acres of the Highly Suitable class. To do this task, the pixel cells for the 200 - Highly Suitable class will be extracted and subsetted to a new vector feature.

To start, Reclass the array of values 0 - 200 into 3 separate bins in the Reclass by Table tool:

Using classes:
1) 0 ≤ x < 199 → 0
2) 200 ≤ x < 200 → 1
3) 201 ≤ x < 300 → 0

Note: the third class is out of the range of values (>200); that’s ok. The tool needs 3 classes to run; its doesn’t matter if a partical range actually exists in the raster layer or not to be a valid range for the tool parameters to be valid.

Next, open the Polygonize tool from the processing toolbox; save the output of this tool to temporary memory:

Warning

Even at 60 meter spatial resolution, the vectorization process and resulting feature is slow. Have patience; both the vectorization and subset for value 200 will take several minutes to process.

Next, Select by Expression the DN value 200; accept output as temporary:

Export the active selection to output as 200_class.shp:

Next, the final SQL selection will be made via Select by Attributes for just those polygons that fit the 800-1000 Acreage criteria for a development site. Once selected, the resulting features will be exported as most_suitable.shp to the exports folder.

Note

You will likely need to Fix Geometry before proceeding to the areal calculation.

To start, acreage needs to be calculated in the attribute table (you can use the Fixed_geometries feature). To do this task, utilize the Add Geometry Attributes tool. The results for the area field will be square meters:

Create a new column named ACRES, type as Double. Use the following calculation for acres in the new column:

"area"/4046.85642

Next, Select by Expression for Acres between 800 and 1000; export this selection to exports directory as most_suitable.shp:

 "ACRES" >= 800 AND "ACRES" <=1000

Note: The results should be a total of 8 selected vector polygon features.

Assignment Step 8 - Preparation for Cartographic Design - Hillshade Surface

With the bulk of the suitability now complete, you will utilize a raster surface function known as hillshade to augment the design process:

Multidirectional hillshade improves terrain visualization. It combines light from six different directions to represent an enhanced visualization of the terrain and to improve the appearance of regions with low relief. It improves the balance between the overexposed and shadow areas of the map. The output is suitable to use as a relief backdrop for topographical, soil, hydrological, land cover, or other thematic maps in which the data will be enhanced with topography.

To start, import the raster layer STRM_DEM_30M.tif. This layer is 30 meter resolution Digital Elevation covering the Assam state boundary:

Search for Hillshade in Processing Toolbox:

Parameters for the Hillshade tool:

  • Input Features: STRM_DEM_30M.tif
  • Z factor: 1 (this is the default)
  • Output raster: exports > hillshade
  • Accept Azimuth and Altitude Defaults
